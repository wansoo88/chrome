# 04. 컴플라이언스 · 정책 체크리스트

> 스토어 심사 리젝션의 90%는 아래 항목 누락. 제출 전 전원 통과 필수.

## 1. Chrome Web Store 정책

### 1.1 Manifest V3 필수 (2024년부터 MV2 지원 종료)
- [ ] `manifest_version: 3`
- [ ] `background.service_worker` 사용 (background.scripts 금지)
- [ ] `host_permissions`는 최소한으로: `["https://x.com/*", "https://twitter.com/*"]`
- [ ] `<all_urls>` 절대 사용 금지 → 즉시 거절

### 1.2 권한 투명성
- [ ] `permissions` 배열의 모든 항목에 대해 스토어 "자세히 설명" 필드에 이유 기입
  - `storage`: "API 키와 퍼소나를 브라우저 로컬에만 저장"
  - `activeTab` (필요 시): "답변 생성 시점의 현재 탭에만 일회성 접근"
- [ ] "Single purpose" 원칙: 확장은 하나의 명확한 목적. 여러 기능 섞으면 거절

### 1.3 개인정보 처리방침 (Privacy Policy)
- [ ] 공개 URL 필수 (스토어 폼에 입력)
- [ ] 다음을 명시:
  - "우리는 사용자 데이터를 수집·전송하지 않는다"
  - "API 키는 브라우저 `chrome.storage.local`에 저장, 외부 서버 미전송"
  - "AI 요청은 사용자 브라우저에서 사용자가 지정한 제공자로 직접 전송된다"
  - "제공자(OpenAI 등) 프라이버시 정책이 적용된다"
- [ ] 호스팅: Vercel/Cloudflare Pages 무료 정적 페이지로 충분
- [ ] **수집=0 선언은 심사 통과율을 크게 높임**

### 1.4 설명 페이지
- [ ] 첫 문장에 핵심 가치 (hook) — 예: "Your voice, your reply. Pay once, bring your own key."
- [ ] 기능 3~5개 bullet
- [ ] 가격 명시 (유료 기능 있을 경우)
- [ ] 스크린샷 1장 이상 (1280×800)

## 2. X(Twitter) 플랫폼 정책

### 2.1 허용 — 우리가 하는 것
- 유저가 **수동으로 클릭**해서 AI가 생성한 텍스트를 **유저가 편집 가능한 상태**로 삽입
- 유저가 **수동으로 게시** — 확장은 게시 버튼 누르지 않음

### 2.2 금지 — 절대 하지 말 것
- ❌ 자동 게시 (post, reply 버튼 프로그램적 클릭)
- ❌ 자동 팔로우/언팔로우/좋아요
- ❌ 대량 동일 내용 게시 (스팸 탐지)
- ❌ X API 사용자 크리덴셜 저장/전송
- ❌ X 데이터의 외부 저장 (트윗을 우리 서버로 보내기 금지)

### 2.3 DOM 조작 원칙
- 유저가 작성 중인 **자신의 입력 필드**에만 텍스트 삽입
- 타인 컨텐츠 수정/숨김 금지

## 3. AI 제공자 (OpenAI / Anthropic / OpenRouter)

### 3.1 BYOK 합법성
- [OpenAI 공식](https://community.openai.com/t/bring-your-own-key-policy/446168)에서 BYOK 허용 확인됨
- 유저가 자신의 계정 키로 자신의 사용량을 부담하는 것 = OpenAI ToS 준수

### 3.2 우리가 지켜야 할 것
- [ ] 키를 **우리 서버로 전송하지 않음** (어기면 OpenAI가 BYOK 인정 취소)
- [ ] 유저에게 "키는 귀하의 브라우저에만 저장됨"을 설치 온보딩에서 고지
- [ ] API 응답/에러를 우리가 로깅하지 않음

### 3.3 AI 생성 컨텐츠 책임
- [ ] 설명 페이지에 "AI가 생성한 컨텐츠의 정확성/적절성은 보장되지 않으며, 게시 전 사용자가 검토해야 함" 명시
- [ ] 금지 주제(혐오/자해/의학/법률 조언) 프롬프트 가드레일은 **제공자 측 기본 가드에 위임** (우리는 과도한 검열 추가하지 않음)

## 4. 결제 · 세금

### 4.1 ExtensionPay 사용 시
- [ ] Stripe 계정 연결 완료 (신원 확인 포함)
- [ ] 환불 정책 명시: "구매 후 14일 이내 이유 불문 환불" (EU 소비자법 디폴트)
- [ ] ExtensionPay 5% 수수료 + Stripe 2.9% + $0.30 = 건당 ~$3.99 기준 약 $0.40 차감
- [ ] 한국 개인사업자 부가세 신고는 분기별 (Stripe → 한국 계좌로 정산 후)

### 4.2 대안 (LemonSqueezy 전환 시점)
- 매출이 월 $2k 넘으면 LemonSqueezy로 전환 검토 (merchant of record = VAT 자동 처리)

- [ ] "Twitter" · "X" 로고/상표 직접 사용 금지. 설명 페이지는 "Works with X (Twitter)" 표기
- [ ] 경쟁사 이름 직접 언급 금지 (비교 마케팅은 별도 블로그에서)

## 6. 제출 직전 최종 체크

- [ ] `manifest.json`의 `version`이 유의미한 숫자 (0.1.0 권장)
- [ ] `description`이 132자 이하 (스토어 제한)
- [ ] 아이콘 16/48/128 전부 포함
- [ ] 프로모 이미지 440×280 (스몰) 필수
- [ ] 스토어 "자세히" 필드에 권한 이유, 원격 코드 사용 여부("No") 기입
- [ ] 테스트 계정 안내 불필요 (BYOK이므로 심사자가 본인 키 사용 가능하나, 스크린샷 기반 심사 요청 시 대비해 데모 영상 준비)
- [ ] 코드에 `console.log` 제거 (글로벌 CLAUDE.md 규칙 + 심사자 체감 품질)
- [ ] 프로덕션 빌드로만 제출 (`pnpm build`)
