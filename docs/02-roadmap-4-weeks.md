# 02. 4주 MVP 로드맵

> 전제: 주 20시간+ 투입. AI 바이브코딩. 총 예산 약 80~100시간.
> 스토어 심사는 신규 개발자 계정 기준 **최대 30일** 소요될 수 있음 → **3주차에 1차 제출**하는 것이 4주 내 출시의 유일한 방법.

---

## 🗓️ Week 0 — 준비 (실제 코딩 전, 0~8시간)

**목표**: 막히지 않도록 계정·결제·기술 준비 끝내기.

- [ ] Chrome Web Store **개발자 등록** ($5 일회성)
- [ ] **ExtensionPay** 계정 생성, 테스트 제품 1개 등록 (무료 샌드박스)
- [ ] Stripe 계정 생성 (ExtensionPay가 Stripe 경유)
- [ ] **OpenAI API 키** 본인용 발급 (개발 중 테스트)
- [ ] Node 20 LTS + pnpm 설치 확인
- [ ] VS Code에 ESLint + Prettier 익스텐션
- [ ] 이 리포지토리 `git init` + GitHub private repo 연결
- [ ] 도메인 구매 (선택이지만 권장, `.com`: $12/년): 스토어 리스팅 신뢰도

**체크리스트 통과 후 Week 1 진입.**

---

## 🗓️ Week 1 — 기반 스캐폴드 + 핵심 플로우 (20~25시간)

**목표**: "X에서 ✨ 버튼 누르면 3안이 나온다"가 로컬에서 끝까지 돈다.

### Day 1~2: 스캐폴드
- [ ] Vite + React + TS + @crxjs 스캐폴드
- [ ] `manifest.json` MV3 설정 (host: x.com, twitter.com)
- [ ] 최소 popup 페이지 "Hello X Reply Booster"
- [ ] `chrome://extensions`에서 로드 확인

### Day 3~4: Content Script 주입
- [ ] x.com에 content script 주입 성공
- [ ] compose textarea 감지 (MutationObserver)
- [ ] ✨ 버튼 주입 (Shadow DOM)
- [ ] 버튼 클릭 → background로 메시지

### Day 5~7: AI 호출 + 팝오버
- [ ] Options 페이지에서 OpenAI 키 입력·저장
- [ ] background service worker에서 OpenAI API 직접 호출
- [ ] 3안 응답 파싱
- [ ] content script에 3안 카드 팝오버 UI
- [ ] "삽입" 버튼으로 textarea에 주입

**Week 1 끝 판정**: 개발자 본인이 x.com에서 실사용 3번 연속 성공.

---

## 🗓️ Week 2 — 퍼소나 + 결제 + 한도 (20~25시간)

**목표**: 제품으로서 완결. 스토어 제출 직전 상태.

### Day 8~10: 퍼소나 시스템
- [ ] Options에 퍼소나 CRUD UI
- [ ] 예시 트윗 10개까지 저장
- [ ] content script 팝오버에 퍼소나 드롭다운
- [ ] 프롬프트 조립 로직 (few-shot)

### Day 11~12: 스레드 힌트
- [ ] textarea에 이미 텍스트 있으면 "threadHint" 모드
- [ ] 빈 텍스트면 "reply" 모드
- [ ] 두 모드 프롬프트 분기

### Day 13~14: 결제 + 한도
- [ ] ExtensionPay 통합
- [ ] 무료: 일 5회 생성 (chrome.storage에 카운터 + 자정 리셋)
- [ ] 6회차 → 업그레이드 모달
- [ ] 결제 성공 → 한도 해제, 퍼소나 슬롯 10개 열림
- [ ] 라이선스 상태 캐싱 + 24시간마다 재검증

**Week 2 끝 판정**: 지인 3명에게 로컬 설치시켜서 버그 없이 동작.

---

## 🗓️ Week 3 — 스토어 제출 + 마케팅 사전 준비 (15~20시간)

**목표**: 스토어 심사 시작 + 론칭 자산 확보.

### Day 15~17: 스토어 리스팅
- [ ] 아이콘 16/48/128px (Figma AI 생성)
- [ ] 스크린샷 5장 (1280×800): 기능별 시연
- [ ] 프로모 이미지 440×280, 1400×560
- [ ] 설명 페이지 본문: hook 문장 → 기능 3개 → 가격 → FAQ
- [ ] 개인정보 처리방침 (간단 정적 페이지, Vercel 무료 호스팅)
- [ ] `docs/04-compliance.md` 체크리스트 전부 통과
- [ ] **스토어 제출** (이 시점부터 심사 타이머 시작)

### Day 18~21: 론칭 자산
- [ ] 랜딩 페이지 (Framer/Carrd, $0~5)
- [ ] X 프로모 글 3종 (티저 / 데모 영상 / 사용 후기 포맷)
- [ ] 데모 GIF 3개 (기능별 10초)
- [ ] Product Hunt 등록 준비 (이메일/사진/카피)
- [ ] 첫 구매자 10명용 할인 코드 (ExtensionPay 쿠폰)

**Week 3 끝 판정**: 스토어 심사 중 + 모든 마케팅 자산 준비 완료.

---

## 🗓️ Week 4 — 심사 대응 + 소프트 런치 (10~20시간)

**목표**: 심사 통과 후 48시간 안에 첫 10명 유료 고객.

### 심사 대응
- [ ] 리젝션 시 사유 대응 (대개 `host_permissions`, `description` 수정)
- [ ] 1차 거절 → 당일 재제출 원칙 (재심사는 보통 3~7일)

### 소프트 런치 (심사 통과 즉시)
- [ ] X에 출시 공지 (본인 계정 + 빌더 커뮤니티)
- [ ] Product Hunt 등록 (화요일~목요일 PST 00:01 출시 권장)
- [ ] IndieHackers 포스트
- [ ] 환불 처리 플로우 테스트 (1건 테스트)
- [ ] 피드백 수집 폼 (Tally.so 무료)
- [ ] 초기 리뷰 수집: 무료 퍼소나 슬롯 1개 추가 보상으로 리뷰 요청

### 상시 모니터링
- [ ] 하루 1회: 스토어 리뷰, 환불 요청, 이메일 확인
- [ ] 주 1회: 매출 집계, 주요 버그 리포트 우선순위화

## 🚦 위험 요소 & 완화

| 위험 | 발생 확률 | 대응 |
|---|---|---|
| Chrome Web Store 심사 2차 거절 | 중 | description에 "BYOK" 명시 + 권한 최소화 + privacy policy URL 넣기 |
| X DOM 구조 변경 | 중 | selector를 aria-label + data-testid 다중 폴백으로 짜기 |
| OpenAI 요금 이슈로 유저 이탈 | 높음 | OpenRouter (GPT-4o-mini 경로 저렴) 지원 빠르게 추가 |
| ExtensionPay 장애 | 낮 | 라이선스 검증 실패 시 grace period 7일 적용 |
| 낮은 평점 조기 발생 | 중 | 첫 주 유저 10명에게 1:1 DM으로 온보딩 도움 제안 |
